# PageBrief

这是一个最小可运行的浏览器插件 MVP：读取当前网页内容，调用用户自己配置的大模型接口，输出一段约 60 到 150 个字的中文摘要。

## 为什么先做这个版本

先把架构想清楚，不要一开始就做“免费模型 + 平台账号体系 + 后端中转”。

这个需求最稳的第一版是：

1. 插件只负责读取当前网页。
2. 用户自己填写 `API URL / Model ID / API Key`。
3. 插件直接从本地请求模型。

这样你不需要后端，也不会把你自己的付费密钥硬编码进插件里。浏览器插件代码是可被用户看到的，直接把开发者密钥放前端是不可行的。

如果你后面要支持“用户不用填 Key，直接免费用”，那就必须再加一层自己的后端服务，用来：

1. 保存你自己的模型密钥。
2. 做登录鉴权。
3. 做限流和配额。
4. 控制滥用和成本。

## 当前实现

当前仓库已经有一个 `Chrome Manifest V3` 插件原型：

- `manifest.json`
- `popup.html`
- `popup.css`
- `popup.js`

功能包括：

1. 在弹窗里配置模型接口。
2. 支持几个常见 OpenAI 兼容接口模板。
3. 自动申请目标 API 域名的运行时网络权限。
4. 读取当前网页正文。
5. 如果用户先选中网页里的文字，会优先总结选中内容。
6. 调用模型生成约 60 到 150 字摘要。
7. 仅在点击保存后，将配置写入当前浏览器本地。
8. 内置调试日志，便于排查 `Failed to fetch`、401、403、429 等问题。
9. 新增 `Ollama` 本地模型模板。
10. 当前版本允许任意 `HTTPS` 的 OpenAI 兼容云端接口，以及本地 `localhost / 127.0.0.1` 接口。

## 在 mac 上怎么跑

1. 打开 Chrome。
2. 进入 `chrome://extensions/`。
3. 开启右上角的开发者模式。
4. 点击“加载已解压的扩展程序”。
5. 选择这个目录：`/Users/lemonstyle/Documents/csr_pj/chrome_summarize`
6. 打开任意网页，点击插件图标。
7. 填写模型配置并保存。
8. 点击“总结当前页”。

## 本地模型怎么填

当前版本支持本地 `OpenAI-compatible API`。

最直接的就是 `Ollama`：

1. 在 mac 上安装并启动 Ollama。
2. 先拉一个本地模型，例如：`ollama pull llama3.2`
3. 在插件里把模板切到 `Ollama 本地模型`
4. 默认接口是：`http://127.0.0.1:11434/v1/chat/completions`
5. `Model ID` 填你本机已经拉下来的模型名，例如：`llama3.2`
6. `API Key` 可以留空

Ollama 官方文档说明，它提供 OpenAI 兼容的 `/v1/chat/completions` 接口，本地场景里 `api_key` 是 required but ignored。

## 怎么看详细报错

现在插件里已经有一块“调试日志”面板，会记录：

1. 配置是否保存成功。
2. 是否拿到了接口域名权限。
3. 页面文本是否提取成功。
4. 模型请求是网络失败还是 HTTP 返回错误。
5. 返回体前 280 个字符预览。

如果你还要更细的日志：

1. 打开 `chrome://extensions/`
2. 打开右上角开发者模式
3. 找到这个插件
4. 点 `Inspect views`
5. 检查 `popup.html`

这样可以直接看弹窗的 Console。

你是 mac 开发也没问题。这个插件逻辑是标准浏览器扩展逻辑，Windows 用户同样可以装。真正决定兼容性的不是你的开发电脑，而是：

1. 你有没有写系统专属能力。
2. 浏览器是否支持相关 API。
3. 你调用的模型接口在目标用户网络环境里能不能访问。

## 你接下来该怎么做

建议按这个顺序推进，不要反过来：

1. 先做 BYOK MVP。
2. 先只支持 `OpenAI-compatible API`，不要一开始写很多家厂商的特殊适配。
3. 先本地加载测试 20 到 30 个网页。
4. 再补异常处理：超时、403、429、空页面、受限页面。
5. 再决定是否要做你自己的后端免费版。
6. 最后再处理商店上架。

## 上架前你要补的东西

这个原型能跑，但如果你准备上架商店，还要补下面这些：

1. 隐私说明页面。
2. 权限用途说明。
3. Logo 和宣传图。
4. 更细的错误提示。
5. 数据处理说明。
6. 如果要做免费版，还要有服务端和用户协议。

## 当前限制

当前代码目前保留这些传输层限制：

1. 云端接口只允许 `HTTPS`。
2. `HTTP` 仅允许本机 `localhost / 127.0.0.1`。
3. 远程域名不再限制具体厂商，只要是 `HTTPS` 的 OpenAI 兼容接口即可。
4. 点击“总结当前页”时，会把当前页面标题、URL 与正文或选中文字发送到用户选择的模型服务。
5. 模型配置、最近摘要与调试日志只保存在当前浏览器本地。

## 适合你的产品路线

如果你现在是一个人开发，最现实的路线是：

1. 先做“用户自带 Key”版。
2. 先让插件把摘要质量、交互和稳定性做对。
3. 等有真实用户以后，再决定是否做“平台代付的免费版”。

这比一开始就背 API 成本和风控负担更稳。
