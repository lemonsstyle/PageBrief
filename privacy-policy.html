PageBrief Privacy Policy

PageBrief 只在用户主动点击“总结当前页”后处理当前网页内容，用来生成一段简短摘要。我们尽量少处理、少存储、少打扰，并把控制权留给用户。

PageBrief only processes page content after the user explicitly chooses to summarize the current page. It is designed to handle as little data as possible, store as little as possible, and keep control with the user.

Overview

你最需要知道的几件事

The essentials

本说明适用于浏览器扩展“PageBrief”。它的单一用途，是在用户主动点击后，对当前网页生成摘要。下面说明插件会处理什么、何时发送数据、数据保存在什么地方，以及用户如何控制这些数据。

This policy applies to the PageBrief browser extension. Its single purpose is to generate a short summary for the current page after the user explicitly asks for it. The sections below explain what data is handled, when it is sent, where it is stored, and how users can control it.

01
Data

插件会处理哪些数据

What data the extension handles

  • 当前网页的标题、URL、描述、正文片段，或用户当前选中的文本。
  • 用户填写的模型配置，包括模型服务模板、API URL、Model ID 与 API Key。
  • 插件本地生成的摘要结果、最近 10 条摘要缓存与调试日志。
  • The current page title, URL, description, body excerpt, or the text currently selected by the user.
  • Model settings entered by the user, including the service preset, API URL, Model ID, and API Key.
  • Locally generated summaries, up to 10 recent summary cache entries, and debug logs.
02
Sharing

数据会在什么时候发送出去

When data is sent

  • 只有当用户主动点击“总结当前页”时，插件才会把网页内容发送到用户选择的模型服务。
  • 插件不会把网页内容发送给本扩展开发者自己的服务器；当前版本不包含开发者中转后端。
  • 如果用户选择的是第三方模型服务，例如 OpenAI、DeepSeek、OpenRouter、SiliconFlow 或其他兼容服务，数据会直接发送到对应服务提供方。
  • 如果用户选择的是本地模型接口，例如 Ollama、LM Studio 或其他本机 OpenAI 兼容服务，数据只会发送到本地地址。
  • 本扩展不会出售用户数据，也不会将用户数据用于广告定向。
  • Page content is sent only when the user explicitly clicks to summarize the current page.
  • The extension does not send page content to any server operated by the developer; the current version has no developer relay backend.
  • If the user chooses a third-party model service, such as OpenAI, DeepSeek, OpenRouter, SiliconFlow, or another compatible provider, the data is sent directly to that provider.
  • If the user chooses a local model endpoint, such as Ollama, LM Studio, or another local OpenAI-compatible service, the data stays on the local address.
  • The extension does not sell user data and does not use user data for advertising or profiling.
03
Storage

数据保存在什么地方，以及用来做什么

Where data is stored and what it is used for

  • 最近摘要与调试日志保存在当前浏览器的本地扩展存储中。
  • 模型配置仅会在用户点击保存配置后写入当前浏览器本地，以便下次继续使用。
  • 其中 API Key 会先在扩展本地加密，再保存到浏览器本地扩展存储中，不以明文形式落盘。
  • 读取当前网页内容，仅用于生成摘要。
  • 保存模型配置，仅用于减少重复输入；保存最近摘要与调试日志，仅用于提升使用体验与排查错误。
  • 开发者不会自动接收这些本地存储内容。
  • Recent summaries and debug logs are stored in the browser's local extension storage.
  • Model settings are written locally only after the user clicks Save Configuration, so they can be reused later.
  • The API key is encrypted inside the extension before it is written to local extension storage, and is not stored in plaintext on disk.
  • Page content is read only for the purpose of generating a summary.
  • Model settings reduce repeated input, while recent summaries and debug logs improve usability and troubleshooting.
  • The developer does not automatically receive any of this locally stored information.
04
Control

用户可以如何管理这些数据

How users can manage this data

  • 用户可以不填写 API Key，仅使用本地模型接口。
  • 用户可以修改或清空本地保存的模型配置。
  • 用户可以在插件界面中清空本地调试日志。
  • 用户可以卸载扩展，以移除浏览器本地扩展存储中的相关数据。
  • Users may leave the API Key blank when using a local model endpoint.
  • Users can edit or clear locally saved model settings.
  • Users can clear local debug logs from the extension interface.
  • Users can uninstall the extension to remove locally stored extension data.
