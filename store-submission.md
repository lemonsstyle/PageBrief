# 商店提交文案模板

这份文件用于准备 `Chrome Web Store` 和 `Microsoft Edge Add-ons` 的提审资料。发布前请把其中的占位信息替换为你的正式内容。

## 产品单一用途

`PageBrief` 的单一用途是：在用户主动点击扩展按钮后，读取当前标签页中可见的网页文本，并调用用户选择的 AI 模型服务生成一段中文摘要。

## 简短描述

在用户主动点击后，为当前网页生成一段中文摘要，支持任意 `OpenAI-compatible` 的 HTTPS 接口，以及本地 OpenAI 兼容接口。

## 详细描述

`PageBrief` 是一个面向中文场景的浏览器扩展。用户点击扩展按钮后，插件会读取当前页面标题、URL 与正文或选中文字，并调用用户选择的 AI 模型服务生成一段简短摘要。当前版本支持任意 `OpenAI-compatible` 的 HTTPS 接口，以及本地 `localhost / 127.0.0.1` 的 OpenAI 兼容接口。模型配置仅在用户点击保存后写入当前浏览器本地；最近摘要与调试日志保存在当前浏览器本地。

## 权限用途说明

### `activeTab`

用于在用户点击扩展后访问当前活动标签页。

### `scripting`

用于在当前页面中提取标题、URL、正文和用户选中文字，以生成摘要。

### `storage`

用于将用户手动保存的模型配置、最近摘要缓存与调试日志保存在当前浏览器本地。

### `optional_host_permissions`

用于在用户选择具体模型服务后，请求对应 AI 接口域名的网络访问权限。当前包含：

- `https://*/*`
- `http://127.0.0.1/*`
- `http://localhost/*`

## 隐私披露建议

### 是否处理网页内容

是。插件会在用户主动点击“总结当前页”后读取当前页面标题、URL、描述、正文片段或选中文字。

### 数据是否会发送到第三方

是。网页内容会被发送到用户自己选择的模型服务提供方，用于生成摘要。

### 数据是否发送给开发者服务器

否。当前版本没有开发者中转后端，网页内容不会自动发送给开发者服务器。

### 本地存储了什么

- 用户手动保存的模型配置
- 用户手动保存的 API URL
- 用户手动保存的 Model ID
- 用户手动保存的 API Key
- 最近 10 条摘要缓存
- 调试日志

### 是否出售用户数据

否。

### 是否用于广告或画像

否。

## 测试说明模板

发布前请在商店后台的 `Test instructions / Notes for certification` 中填写：

1. 打开任意公开文章页，例如新闻页、博客页或产品介绍页。
2. 点击扩展图标。
3. 在 `模型配置` 中选择一个受支持的服务模板。
4. 填入审核专用测试用 `API URL / Model ID / API Key`。
5. 点击 `保存配置`。
6. 点击 `总结当前页`。
7. 插件会显示一段中文摘要，并在重新打开弹窗后恢复该页面最近一次摘要。

如果你不准备给审核员提供云端测试 key，就需要在这里明确写明如何使用本地模型进行审核，例如 `Ollama` 的安装与测试模型名。
